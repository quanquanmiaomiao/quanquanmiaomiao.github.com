字符编码

和编码有关的操作

String.prototype.codePointAt
返回一个字符的码点(该码点是十进制的),并且也能正确返回4个字节字符串的码点

String.fromCodePoint
返回码点对应的字符

String.prototype.normalize
为了表示语调和重音符号，Unicode提供了两种方式:
方式一:直接提供带重音符号的字符 (如 Ǒ (\u01D1)）
方式二:提供合成符,号即原字符与重音符号的合成 (如 O（\u004F）和ˇ（\u030C）合成Ǒ（\u004F\u030C)）
‘\u01D1’===’\u004F\u030C’ 意义是等价的但是JS不能识别

ES6提供了String.prototype.normalize方法，将字符的不同表示方法统一为同样的形式，称为Unicode的正规化

normalize可以接受四个参数
NFC:默认参数，表示“标准等价合成”(视觉和意义上的等价)，返回多个简单字符的合成字符
NFD:表示“标准等价分解”，即在标准等价的前提下，返回合成字符分解的多个简单字符
NFKC:表示“兼容等价合成”(语义上等价，如“囍”和“喜喜” 视觉不等价但语义是等价的)，返回合成字符
NFKD:表示“兼容等价分解”，即在兼容等价的前提下，返回合成字符分解的多个简单字符

注意:normalize不能识别三个或三个以上字符合成的字符串。
